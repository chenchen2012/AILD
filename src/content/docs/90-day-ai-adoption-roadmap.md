## Phase 1 (Days 1-30): pilot foundations

- select 1-3 high-frequency workflows
- deploy policy and QA controls
- track baseline and pilot metrics

## Phase 2 (Days 31-60): functional expansion

- add two adjacent workflows per function
- train managers on quality and risk gates
- create shared prompt and output standards

## Phase 3 (Days 61-90): operating system

- standardize governance meetings
- publish role-based playbooks
- implement KPI dashboard for leadership review

## Adoption KPI stack

- active workflow usage
- time saved per workflow
- quality pass rate
- rework reduction
